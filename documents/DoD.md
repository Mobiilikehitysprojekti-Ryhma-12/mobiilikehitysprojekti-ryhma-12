# Definition of Done (DoD)

QuoteFlow-projektin yhteinen Definition of Done kaikille tehtäville.

---

## Tehtävä on valmis kun:

✅ **Koodi on kirjoitettu**
- Toiminnallisuus toteutettu täysin
- Koodi noudattaa projektin koodausstandardeja
- Ei debug-koodia tai kommentoitua testikoodia

✅ **Koodi on versionhallinnassa**
- Commitoitu omaan branchiin
- Commit-viestit selkeitä ja kuvaavia
- Pushattu GitHubiin

✅ **Code review tehty**
- Pull request luotu
- Vähintään yksi tiimiläinen tarkastanut koodin
- Mahdolliset kommentit käsitelty
- PR hyväksytty ja mergetty mainiin

✅ **Testattu**
- Toiminnallisuus testattu manuaalisesti
- Testattu sekä onnistumis- että virhetilanteet
- Testattu eri laitteilla/emulaattoreilla

✅ **Ei kriittisiä bugeja**
- Sovellus ei kaadu
- Ei datahävikkiä
- Ei käyttöä estäviä virheitä

✅ **Dokumentaatio**
- README päivitetty tarvittaessa
- Kommentit koodissa missä tarpeen
- API-muutokset dokumentoitu

✅ **Issue suljettu**
- GitHub issue päivitetty
- Siirretty Done-sarakkeeseen kanban-boardilla
- Linkattu PR:ään

---

## Sprint-kohtainen DoD

**Sprint demo varten:**
- Toiminnallisuus demottavissa
- Demo-skenaario testattu
- Ei tunnetut bugit demon aikana

**Sprint päättyessä:**
- Kaikki sprint-tehtävät täyttävät DoD:n
- Sprint retrospektiivi tehty
- Seuraavan sprintin backlog priorisoitu
